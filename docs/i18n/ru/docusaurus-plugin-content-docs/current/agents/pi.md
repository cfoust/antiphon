# Pi

API расширений Pi — самый полный из всех: этот адаптер достигает полного паритета с
Claude Code, включая настоящий обратный канал — сообщения из комнаты приходят в сессию как
подлинные пользовательские сообщения через `pi.sendUserMessage`.

## Установка

```bash
cp plugins/pi/index.ts ~/.pi/agent/extensions/antiphon.ts
```

Горячая перезагрузка — командой `/reload`.

## Что вы получаете

- **Привязка** на `session_start`, с живыми заголовками.
- **Тики инструментов** на `tool_execution_start`.
- **Нарратив** — адаптер регистрирует `antiphon_task` / `antiphon_progress` /
  `antiphon_done` / `antiphon_blocked` как нативные инструменты и добавляет мандат
  нарратива к системному промпту.
- **Подстраховка «готово»** — `agent_end` проговаривает финальное сообщение ассистента,
  если модель не рассказала свой итог сама.
- **Обратный канал, по-настоящему** — кадры канала из хаба становятся пользовательскими
  сообщениями в сессии с тегом `<channel source="antiphon">`, так что вы можете ответить
  агенту из комнаты, и он немедленно примет это к действию.

Одно ограничение: Pi не открывает расширениям событие запроса разрешений, поэтому сигналы
блокировки звучат только тогда, когда модель сама вызывает `antiphon_blocked`.
