# Claude Code

Эталонная интеграция — и самая полная: привязка идентичности, тики инструментов, нарратив,
сочиняемый LLM, устные итоги, уведомления о блокировке и обратный канал из комнаты в
сессию.

## Установка

```bash
claude plugin marketplace add cfoust/antiphon
claude plugin install antiphon@antiphon
```

Это всё. Плагин находит `antiphond` сам (тот встроен в `Antiphon.app`) и начинает
озвучивать работу уже со следующей сессии.

## Как это устроено

- MCP-сервер (`antiphond channel`) даёт модели четыре инструмента — `antiphon_task`,
  `antiphon_progress`, `antiphon_done`, `antiphon_blocked`, — а хук на старте сессии
  вставляет короткий мандат: думать вслух через них. Нарратив, который вы слышите, модель
  сочиняет сама по ходу работы.
- Хук `PostToolUse` шлёт тик по принципу «выстрелил и забыл» на каждый вызов инструмента —
  это и есть аккорд.
- Сообщения, отправленные из комнаты, приходят в сессию как пользовательский ввод.

## Разработка против рабочей копии

```bash
claude --plugin-dir /path/to/antiphon/plugins/claude-code
```

Если вы работаете над самим демоном, укажите конкретный его бинарник через переменную
`ANTIPHOND`.
