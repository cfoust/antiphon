# 隐私

Antiphon 是本地优先的应用。没有账号，没有遥测，没有分析统计——而所有可能敏感的部分，
从设计上就留在你的机器上。

## 摄像头

头部追踪完全在设备上运行：原生应用把画面交给 Apple 的 Vision 框架，网页演示交给浏览器里的
MediaPipe。视频从不录制、从不保存、从不传输——画面在内存中分析后即被丢弃。当你合上
Antiphon 的眼睛（菜单栏开关），采集会话彻底停止，摄像头指示灯熄灭。

## 什么会离开你的机器

只有一件事，而且只在你主动开启时发生：**语音合成**。如果在"设置 → 声音"里添加了
ElevenLabs 或 OpenAI 的 API 密钥，智能体播报的*文本*会发送给该提供商合成语音，适用其
隐私政策。合成的音频缓存在本地（`~/.antiphon/tts-cache`），重复的台词不会再次发送。
如果使用内置的 macOS 声音，则没有任何数据离开你的机器。

另有两个小的例行请求：应用每天向 GitHub 的公开 API 询问一次是否有新版本（一个普通的
HTTPS 请求——除了任何网页请求都会携带的信息外，不含任何标识符；可在"设置 → 关于"里关闭），
网站上的下载按钮也做同样的事。网络故事到此为止。

`antiphond` 守护进程只监听 `127.0.0.1`——从网络上无法访问。智能体会话的播报从插件到
守护进程再到应用，全程走 localhost。

## 磁盘上有什么

Antiphon 存储的一切都在 `~/.antiphon`：

- `config.json` — 设置，包括 API 密钥（文件权限 `0600`）
- `agents.json` — 智能体注册表（会话 id、角色、座位）
- `tts-cache/` — 合成的音频
- `antiphond.log` — 守护进程的诊断日志，每次运行清空

删掉这个目录，Antiphon 就把你忘了——见[卸载](./install.md#卸载)。

## 这个网站

网站是静态的（GitHub Pages）。下载按钮会查询 GitHub 的公开 API 来找到最新版本；网页
演示完全在你的浏览器里运行，不向任何地方发送数据。
