不知從什麼時候開始，與程式代理（coding agent）一起工作，變成了盯著它們看。如今一場順利的工作階段，意味著四五個代理同時在跑——一個在重構、一個在寫文件、一個在追一個時好時壞的測試——而我，像巡查監視器的警衛一樣，在終端機分頁之間輪轉。代理一個月比一個月更快、更能幹。我這一邊的分工卻始終如一：輪詢、瞇眼、捲動。

仔細想想，這是一種很奇怪的協作監督方式。沒有人靠同時盯著所有人來帶團隊。你們共享一個空間。你培養出餘光般的覺察。你聽得出房間的氛圍——誰進入了心流，誰卡住了，誰剛推開椅子、有話要說。視覺是前景的感官；它需要你去對準。聽覺才是背景的那一個。它無時無刻監聽著一切，而且分文不取。

於是我做了那個我想要的東西：一個房間。

<Eye />

Antiphon 給每個代理工作階段一個聲音，以及你周圍虛擬空間中的一個座位。它是一個原生 macOS 應用程式，做的是真正的雙耳（binaural）渲染——實測的 HRTF、室內聲學，還有一台看著你頭部的網路攝影機，讓你移動時世界保持靜止。代理不再是一個分頁；它是離你左肩兩公尺的一個存在。它執行工具時，你會聽到它的和弦一個音一個音向下滴答。它沉入一場漫長的建置時，會落進低沉的三音機械嗡鳴，就像隔著工作坊聽見一台車床。它完成時，會告訴你——一兩句話，用它自己的聲音，從它在房間裡自己的位置。

我最在乎的互動，是那個什麼都不用付出的互動：**「閉上眼睛」。** 攝影機會察覺。書桌淡去，房間浮現，有那麼幾秒鐘，你就站在你的代理之間。轉向那個需要你的聲音，凝望一拍，它便交出它的報告。睜開眼睛，回覆框已在等候——輸入、按下 Enter，你又回到自己的工作。用耳朵分流，十秒鐘，零分頁。

> 沒有任何提示音，沒有任何閃爍。等待中的代理只會在你的一側耳中，緩慢地醞釀出一次綻放——以分鐘計——直到你有空理它。

這種溫柔正是設計的中心，也是我花了最多力氣重新學習的部分。我用過的每一套通知系統，都把注意力當成要*奪取*的東西。Antiphon 把它當成你*出借*的東西。這些聲音經過數十輪的以耳調校才調對——更安靜、更低、更慢，永遠朝著那個方向——因為環境式工具的失敗模式不是太含蓄，而是惹人厭。如果你注意到了音效設計的存在，它就是錯的。

## 工程上的賭注

引擎蓋底下只有一個賭注，而且下得很早：單一 Rust DSP 核心，沒有 I/O、沒有執行緒、音訊路徑上不做記憶體配置，編譯兩次——一份是 Mac 應用程式裡的 staticlib，一份是給瀏覽器的 WebAssembly 模組。兩份建置被驗證為逐位元組相同，這也是為什麼[網頁展示](https://antiphon.dev/demo.html)不是產品的樣品屋；它*就是*那個渲染器，只是跑的是照劇本走的代理，而不是你的。連首頁上的示範影片，也是離線經由真正的引擎配樂的。

另一件比預期重要得多的事：頭部追蹤勝過 HRTF 個人化。人人都愛問精緻的逐耳量測，但真正讓大腦相信聲音就在外面的是運動——聲場快速而平滑地隨你的頭反向旋轉。一支貼合滑桿加一台網路攝影機，就能帶你走完幾乎全程。

一個小小的 Go 常駐程式把一切繫在一起：代理外掛連上 localhost，工作階段獲得固定的角色與聲音（有金鑰就用 ElevenLabs 或 OpenAI，沒有就用 macOS 內建語音），一切串流到應用程式。Claude Code、Codex、OpenCode、Pi 與 Aider 今天都有轉接器，而協定刻意保持精簡——寫一個轉接器，不過是一個下午的事。

## 名字

*Antiphon* — ἀντίφωνον（對唱之歌）——是「應答輪唱」的古字：聲音在空間中彼此應答。這正是如今工作的形狀，而它值得比儀表板更好的東西。整個專案[開放原始碼，MIT 授權](https://github.com/cfoust/antiphon)。

戴上耳機。[閉上眼睛。](https://antiphon.dev/)
