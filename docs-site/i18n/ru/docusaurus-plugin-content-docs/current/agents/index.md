# Подключение агентов

Antiphon слушает сессии агентов через небольшие адаптеры, которые общаются с локальным
демоном `antiphond`. Каждый адаптер — **fail-open**: если Antiphon не установлен или не
запущен, агент ведёт себя ровно так, как будто плагина не существует. Ничто и никогда не
блокирует сессию из-за демона.

## О чём сообщает адаптер

| Событие | Что вы слышите |
| --- | --- |
| Сессия началась | Агент занимает место в комнате |
| Вызов инструмента | Одна нота аккорда агента |
| Нарратив | Короткие реплики о ходе работы от первого лица, вслух |
| Готово | Итог в одно-два предложения, произнесённый с его места |
| Заблокирован / ждёт вас | Медленное цветение ожидания в одном ухе |

Полнота зависит от того, что открывает API расширений конкретного агента, — точная матрица
поддержки есть на странице каждого из них.

## Здесь мог бы быть ваш агент

Протокол намеренно мал: WebSocket на `127.0.0.1` (`/agent`) или одноразовые события через
`antiphond emit`. Прочитайте любой адаптер в
[`plugins/`](https://github.com/cfoust/antiphon/tree/main/plugins) — самый маленький
умещается в один файл. PR приветствуются.
