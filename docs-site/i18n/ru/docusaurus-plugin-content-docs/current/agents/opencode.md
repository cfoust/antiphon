# OpenCode

Внутрипроцессный API плагинов OpenCode позволяет этому адаптеру добиться высокой полноты
одним TypeScript-файлом без зависимостей: живая привязка сессий с заголовками, тики
инструментов, нарратив от модели через зарегистрированные инструменты `antiphon_*`,
подстраховка на простое, проговаривающая последнее сообщение ассистента, и сигналы
блокировки из событий разрешений.

## Установка

```bash
cp plugins/opencode/index.ts ~/.config/opencode/plugins/antiphon.ts
```

Это вся установка. (Публикация npm-пакета для массива `plugin` в `opencode.json`
запланирована; файл работает одинаково в обоих случаях.)

## Что вы получаете

- **Привязка** на `session.created` — сессия занимает место, заголовок берётся из вашего
  промпта.
- **Тики инструментов** на каждый `tool.execute.after`.
- **Нарратив** — адаптер регистрирует четыре инструмента `antiphon_*` и вставляет мандат
  нарратива в системный промпт, так что модель думает вслух от первого лица.
- **Подстраховка «готово»** — если модель не рассказала сама, `session.idle` проговаривает
  последнее сообщение ассистента (с дедупликацией против вызванного моделью
  `antiphon_done`).
- **Блокировка** на `permission.asked`.
- **Обратный канал** — сообщения из комнаты доставляются в сессию через OpenCode SDK
  (по мере возможности), с инъекцией в панель терминала как запасным путём.

Дочерние сессии сабагентов молчат — места получают только ваши сессии верхнего уровня.
