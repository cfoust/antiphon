# Движок

Рендерер Antiphon — бинауральный движок, построенный специально под эту задачу, а не
игровое аудио-middleware. Конструктивное ограничение, определившее всё остальное:
**одно DSP-ядро на Rust, два хоста, побайтово идентичный вывод.**

## Одно ядро, две цели

Движок (`antiphon-dsp`) — один Rust-крейт без ввода-вывода, без потоков и без аллокаций
на аудиопути. Наружу он открыт через единый C ABI (`antiphon-ffi`), который компилируется
двумя способами:

- **staticlib**, слинкованная в нативное приложение macOS и опрашиваемая
  `AVAudioSourceNode`;
- **wasm32 cdylib**, которой управляет `AudioWorklet` в браузере.

Стенд паритета рендерит одну и ту же сцену обоими путями и проверяет, что выводы совпадают.
Они совпадают — с точностью около −155 dBFS, а это значит: веб-демо *и есть* рендерер
приложения.

## Тракт сигнала

На каждый голос, на каждый блок из 128 фреймов при 48 кГц:

1. **HRTF-свёртка** — минимально-фазовые HRIR (128 отводов, SIMD) из измеренного набора
   MIT KEMAR, с отдельной линией дробной задержки для ITD, чтобы межушная синхронизация
   оставалась точной при движении головы.
2. **Дистанция** — уровень, поглощение в воздухе и коррекция ближнего поля.
3. **Ранние отражения** — модель мнимых источников второго порядка для прямоугольной
   комнаты с частотно-зависимым поглощением каждой поверхности; для каждого источника
   сохраняются самые громкие образы в рамках бюджета, ранжированного по энергии.
4. **Поздняя реверберация** — либо сеть задержек с обратной связью на 16 линий, либо
   секционированная BRIR-свёртка, в зависимости от пресета комнаты. Комната по умолчанию —
   зал, рендеримый свёрткой.
5. **Трекинг головы** — поза слушателя вращает всё звуковое поле; коэффициенты плавно
   меняются от блока к блоку, поэтому движение обходится без щелчков.

Голоса могут также нести диаграмму направленности (частотно-зависимая кардиоида, корректно
зеркалируемая на отражения) и объёмную протяжённость (декоррелированные сателлитные отводы
на тетраэдре) — оба механизма, пока не используются, являются битово-точными no-op.

## Почему трекинг головы важен

Статичный бинауральный звук норовит схлопнуться «внутрь головы», особенно для источников
спереди. Сильнейшее противоядие — не персонализированная HRTF, а **движение**: когда мир
контрвращается относительно вашей головы с низкой задержкой, мозг принимает
экстернализацию. Поэтому приложение отслеживает голову веб-камерой, и поэтому ползунок
подстройки (простое частотное масштабирование HRTF) — единственная ручка персонализации:
на практике основную работу делает трекинг.

## Офлайн-оракул

Всё, что можно услышать, проходит через `antiphon-render` — офлайн-рендерер, который делает
WAV-файлы из описаний сцен. Он служит и порогом качества (изменения оцениваются на слух,
в наушниках), и оракулом паритета. Даже саундтрек демо на маркетинговом сайте сгенерирован
через него — рекламу продукта рендерит сам продукт.
